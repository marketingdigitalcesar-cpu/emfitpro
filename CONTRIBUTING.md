# Guía de Contribución 🤝

¡Gracias por interesarte en mejorar **Emfitpro**! Para mantener la calidad del proyecto, por favor sigue estas guías:

## 📜 Formato de Commits

Usamos un estándar para los mensajes de commit para que el historial sea legible:

- `feat:` Una nueva funcionalidad (ej: `feat: implementación de Wompi`)
- `fix:` Corrección de un error (ej: `fix: error en login de Google`)
- `docs:` Cambios solo en la documentación (ej: `docs: actualizar readme`)
- `refactor:` Un cambio de código que ni corrige un error ni añade una funcionalidad (ej: `refactor: mejorar estructura de config.php`)

## 🛠️ Proceso de Desarrollo

1. **Sincroniza**: Asegúrate de tener la última versión de la rama `main`.
2. **Ramas**: Usa nombres descriptivos para tus ramas, ej: `fix/error-webhook` o `feat/nueva-pantalla`.
3. **Pruebas**: Antes de enviar tus cambios, asegúrate de que no rompen el sistema de login ni el pago.

## 🐛 Reportar Errores

Si encuentras un error, abre un "Issue" en GitHub detallando:
- Pasos para reproducirlo.
- Comportamiento esperado vs real.
- Screenshots si es posible.
